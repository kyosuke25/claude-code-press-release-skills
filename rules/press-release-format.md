# プレスリリース フォーマットルール

## 構成（必須順序）

プレスリリースは以下の順序で構成すること：

| 順番 | 要素 | 必須 | 備考 |
|------|------|------|------|
| 1 | ヘッダー | ○ | 日付（YYYY年M月D日）、会社名を右寄せ |
| 2 | タイトル | ○ | 【サービス名】〇〇します/しました |
| 3 | サブタイトル | ○ | 〜補足説明〜 の形式 |
| 4 | リード文 | ○ | 定型の書き出しから始める |
| 5 | カバー画像 | △ | リード文の後に配置 |
| 6 | 本文 | ○ | ■見出しで区切る |
| 7 | 関連情報 | △ | 提携先情報、ロードマップ等 |
| 8 | 会社概要 | ○ | 定型文を使用 |
| 9 | 問い合わせ先 | ○ | 定型文を使用 |

## タイトルの書き方

**形式：** `【サービス名】〇〇します` または `【サービス名】〇〇しました`

✅ 良い例：
- 【サービス名】法人プランを開始します
- 【サービス名】正式版をリリースしました
- 【サービス名】新機能を追加しました

❌ 悪い例：
- サービス名についてのお知らせ（【】がない）
- 【サービス名】についてのご案内（動詞が曖昧）
- 新サービス開始！（サービス名がない）

## サブタイトルの書き方

**形式：** `〜補足説明〜`

✅ 良い例：
- 〜福利厚生や社員教育の充実のためにご利用いただけます〜
- 〜新機能で業務効率が向上します〜

❌ 悪い例：
- 補足説明（〜で囲まれていない）
- ～半角チルダ～（全角〜を使用すること）

## リード文の書き方

**必ず以下の形式で開始する：**

```
〇〇を運営する[会社名]（本社：[所在地]、代表取締役：[代表者名]、以下「当社」）は、このたび〇〇することをお知らせします。
```

※ 会社情報は `.claude/rules/company-info.md` を参照

**ポイント：**
- 1文目で「誰が」「何を」「いつ」を明確にする
- 2文目以降で背景や詳細を補足する
- リード文全体は3〜5文程度に収める

## 文体ルール

| ルール | 詳細 | 例 |
|--------|------|-----|
| 敬体 | ですます調で統一 | 〜します、〜いたします |
| 敬語 | ビジネス敬語を使用 | いたします、ございます |
| 文長 | 一文60〜80文字目安 | 長すぎる文は分割 |
| 専門用語 | 初出時に説明を付ける | NFT（Non-Fungible Token:非代替性トークン） |
| 略語 | 初出時にフルスペル記載 | MVP（Minimum Viable Product） |

## 禁止事項

- [ ] 「業界初」「画期的」等の誇張表現（裏付けがある場合のみ可）
- [ ] 競合他社への言及・批判
- [ ] 未確定情報の断定的記載（「予定」「見込み」を使う）
- [ ] 絵文字の使用
- [ ] 口語表現（「〜だと思います」「〜かもしれません」）

## レビュー時のチェックリスト

- [ ] タイトルは【サービス名】形式になっているか
- [ ] サブタイトルは〜で囲まれているか
- [ ] リード文は定型の書き出しで始まっているか
- [ ] 5W1H（誰が・何を・いつ・どこで・なぜ・どのように）が明確か
- [ ] 専門用語に説明があるか
- [ ] 日付・数字・URLに誤りがないか
- [ ] 会社概要・問い合わせ先があるか
- [ ] 禁止事項に該当する表現がないか
